##################
AST Usage Guide
##################

This guide explains how to work with the PSS Abstract Syntax Tree (AST)
generated by pssparser.

.. contents:: Table of Contents
   :local:
   :depth: 2

.. note::
   The symbol scope tree returned by :meth:`~pssparser.Parser.link` wraps
   the raw parse-time AST nodes.  Most analysis work should be done on the
   **linked** tree rather than the raw ``GlobalScope`` objects.

***************
Getting Started
***************

Basic Parsing
=============

To parse PSS code and obtain the linked symbol tree:

.. code-block:: python

    from pssparser import Parser

    p = Parser()
    p.parses([
        ("inline.pss", """
            component pss_top {
                action my_action {
                    rand int<8> value;
                    constraint { value >= 10; value <= 100; }
                }
            }
        """)
    ])
    root = p.link()
    print("Root type:", type(root).__name__)   # RootSymbolScope

Understanding the AST Structure
================================

The PSS AST follows a hierarchical structure:

* **GlobalScope** — root of each parsed file; one per ``parse``/``parses`` call
* **PackageScope** — package declarations
* **TypeScope** — types (Action, Component, Struct, …)
* **Scope** — any node that can contain children
* **ScopeChild** — base class for most AST nodes

After :meth:`~pssparser.Parser.link`, the tree becomes a *symbol scope* tree
that mirrors the structure above but adds resolved symbol tables:

.. code-block:: text

    RootSymbolScope
    └── SymbolTypeScope  (component pss_top)
        └── SymbolTypeScope  (action my_action)
            ├── Field  (rand int<8> value)
            └── ConstraintBlock
                └── ConstraintStmtExpr  (value >= 10)

********************
Traversing the AST
********************

Navigating by name
==================

``RootSymbolScope`` (and all ``SymbolChildrenScope`` nodes) expose a
symbol table that maps names to child indices:

.. code-block:: python

    from pssparser import Parser

    p = Parser()
    p.parses([("t.pss", "component pss_top { action Write {} action Read {} }")])
    root = p.link()

    # Look up 'pss_top'
    if root.symtabHas("pss_top"):
        idx = root.symtabAt("pss_top")
        top = root.getChild(idx)
        print("pss_top found:", type(top).__name__)

        # Look up 'Write' inside pss_top
        if top.symtabHas("Write"):
            write = top.getChild(top.symtabAt("Write"))
            print("Write:", type(write).__name__)

Iterating children
==================

.. code-block:: python

    for i in range(root.numChildren()):
        child = root.getChild(i)
        print(i, type(child).__name__)

Using SymbolScopeUtil for qualified names
==========================================

:class:`pssparser.utils.SymbolScopeUtil` provides a convenient
``getQname()`` helper:

.. code-block:: python

    from pssparser import Parser
    from pssparser.utils import SymbolScopeUtil

    p = Parser()
    p.parses([("t.pss", """
        package mypkg {
            component pss_top {
                action Write {}
            }
        }
    """)])
    root = p.link()

    util = SymbolScopeUtil(root)
    write = util.getQname("mypkg::pss_top::Write")
    print("Write:", type(write).__name__)

Simple node-type inspection
============================

Use Python's ``type()`` or ``isinstance()`` checks against classes from
``pssparser.ast``:

.. code-block:: python

    import pssparser.ast as ast

    node = root.getChild(0)
    if isinstance(node, ast.SymbolTypeScope):
        # node.getTarget() returns the underlying TypeScope AST node
        target = node.getTarget()
        print("Target type:", type(target).__name__)

Recursive walk
==============

.. code-block:: python

    def walk(node, depth=0):
        indent = "  " * depth
        print(f"{indent}{type(node).__name__}")
        for i in range(node.numChildren()):
            walk(node.getChild(i), depth + 1)

    walk(root)

*********************
Working with Expressions
*********************

Expression Types
================

PSS expressions follow a class hierarchy:

* **Expr** — base class for all expressions
* **ExprBin** — binary operations (``a + b``, ``x > y``)
* **ExprUnary** — unary operations (``!flag``, ``-value``)
* **ExprId** — simple identifiers
* **ExprNumber** — numeric literals
* **ExprBool**, **ExprString** — other literals

Describing an expression
=========================

.. code-block:: python

    import pssparser.ast as ast

    def describe_expr(expr):
        if isinstance(expr, ast.ExprNumber):
            return f"Number({expr.getVal()})"
        elif isinstance(expr, ast.ExprBool):
            return f"Bool({expr.getVal()})"
        elif isinstance(expr, ast.ExprId):
            return f"Identifier({expr.getId()})"
        elif isinstance(expr, ast.ExprBin):
            lhs = describe_expr(expr.getLhs())
            rhs = describe_expr(expr.getRhs())
            return f"({lhs} {expr.getOp()} {rhs})"
        else:
            return type(expr).__name__

*************************
Working with Activities
*************************

Activity Structure
==================

Activities define the control flow of actions:

* **ActivitySequence** — sequential execution (default)
* **ActivityParallel** — parallel execution with join
* **ActivitySchedule** — flexible ordering
* **ActivitySelect** — random branch selection
* **ActivityRepeatCount**, **ActivityRepeatWhile** — loops
* **ActivityIfElse** — conditionals

.. code-block:: python

    import pssparser.ast as ast

    def describe_activity(node, depth=0):
        indent = "  " * depth
        name = type(node).__name__
        print(f"{indent}{name}")
        for i in range(node.numChildren()):
            describe_activity(node.getChild(i), depth + 1)

*************************
Working with Constraints
*************************

Constraint Types
================

* **ConstraintStmtExpr** — expression constraints (most common)
* **ConstraintStmtForeach** — array element constraints
* **ConstraintStmtIf** — conditional constraints
* **ConstraintStmtImplication** — implication constraints (``->``)
* **ConstraintStmtUnique** — uniqueness constraints

***************************
PSS to AST Mapping Examples
***************************

Action Declaration
==================

PSS Code:

.. code-block:: none

    action my_action {
        rand int<8> value;

        constraint {
            value >= 10;
        }

        activity {
            do_something();
        }
    }

Linked symbol tree:

.. code-block:: text

    SymbolTypeScope  (name="my_action")
    └── target → Action
        ├── Field (name="value", type=DataTypeInt, attr=Rand)
        ├── ConstraintBlock
        │   └── ConstraintStmtExpr
        │       └── ExprBin (op=BinOp_Ge)
        │           ├── ExprId ("value")
        │           └── ExprNumber (10)
        └── ActivityDecl
            └── ActivitySequence
                └── ActivityActionHandleTraversal
                    └── ExprRefPath ("do_something")

Parallel Activity
=================

PSS Code:

.. code-block:: none

    parallel {
        action1;
        action2;
    }

AST Structure:

.. code-block:: text

    ActivityParallel
    ├── ActivityActionHandleTraversal ("action1")
    └── ActivityActionHandleTraversal ("action2")

Advanced Topics
**************

Resolving symbol references
============================

Many AST nodes contain :class:`pssparser.ast.SymbolRefPath` objects.
Use :func:`pssparser.core.resolveSymbolPathRef` to resolve them to the
actual declaration node:

.. code-block:: python

    import pssparser.core as core

    resolved = core.resolveSymbolPathRef(root, some_node.getTarget())
    if resolved is not None:
        print("Resolved to:", type(resolved).__name__)

Getting the target AST node from a symbol scope
================================================

Symbol scope nodes (``SymbolTypeScope``, ``SymbolScope``, etc.) wrap the
original parse-time AST node.  Access it with ``.getTarget()``:

.. code-block:: python

    top_sym = root.getChild(root.symtabAt("pss_top"))
    target  = top_sym.getTarget()   # returns the Component AST node
    print("Component name:", target.getName().getId())

Custom visitor (C++ style)
===========================

For exhaustive traversal you can implement ``VisitorBase`` in C++ (see
:doc:`integrator_guide`).  From Python, duck-typing with ``isinstance``
checks is usually sufficient for most analysis tasks.

**********
Best Practices
**********

1. **Work on the linked tree** — use the ``RootSymbolScope`` from
   ``Parser.link()``, not the raw ``GlobalScope`` objects, for any
   name-based analysis.
2. **Check node types** — use ``isinstance()`` before accessing
   type-specific attributes.
3. **Handle optional fields** — many AST nodes have optional children
   (e.g., the ``init`` expression on a ``Field``).
4. **Understand scoping** — the ``parent`` pointer links child to parent;
   ``numChildren()`` / ``getChild()`` links parent to children.
5. **Preserve source locations** — when constructing new nodes, carry
   ``Location`` information for downstream error reporting.
